ЗАДАЧА: Отчет по утилизации хард-ресурсов кластера (CPU, Память, Диск)

ИСХОДНЫЕ ДАННЫЕ:
- CPU утилизация по узлам (в процентах)
- Память по узлам (в процентах от общего)
- Диск по PVC(в процентах)

РЕКОМЕНДУЕМЫЕ ПОРОГИ (ориентиры):
- CPU: норма < 70%; предупреждение 70–85%; критично > 85%
- Память: норма < 75%; предупреждение 75–85%; критично > 85%
- Диск: норма < 70%; предупреждение 70–85%; критично > 85%

ТРЕБОВАНИЯ К АНАЛИЗУ:
1) Проанализируйте все узлы по CPU/Memory и все PVC по Disk на основе указанных рядов.
2) Для каждого ресурса укажите минимум/среднее/максимум, peak_time и характерные окна (start–end) при превышении порогов.
3) Сопоставьте пики ресурсов с окнами деградаций в других доменах (если заметны по времени).
4) Определите узкие места: перегруженные узлы, риск исчерпания памяти/диска, неравномерность распределения.

СТРУКТУРА ОТВЕТА (ВСЕ ТЕКСТОВЫЕ ПОЛЯ — НА РУССКОМ):
- Строго JSON: {verdict, confidence, findings[], recommended_actions[], affected_components[]}.
- Поле verdict ДОЛЖНО принимать только одно из: "Успешно" | "Есть риски" | "Провал" | "Недостаточно данных".
- Для каждого findings укажите: severity (critical/high/medium/low), component (node|persistentvolumeclaim), evidence с интервалом start–end и peak_time.
- Не придумывайте отсутствующие данные; при недостаточности верните verdict="Недостаточно данных".

ПРИМЕЧАНИЯ:
- component извлекайте из лейблов node|persistentvolumeclaim; если нет — используйте "unknown".
- severity определяйте по порогам выше; учитывайте длительность окна и масштаб влияния.
